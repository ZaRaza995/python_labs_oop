# Отчет по ЛР-2: Коллекция объектов

Реализован контейнерный класс `Hospital` (Больница) для управления множеством объектов «Пациент».

## Процесс и ответы на вопросы

### Вопрос 1. Что такое контейнер и чем он отличается от сущности?
**Сущность (`Patient`)** — это описание одного конкретного объекта (чертеж). 
**Контейнер (`Hospital`)** — это объект, который хранит группу сущностей и предоставляет методы для управления ими (добавление, поиск, удаление). Контейнер инкапсулирует внутри себя список `_patients`, не давая изменять его напрямую без проверок.

### Вопрос 2. Как обеспечена целостность данных в коллекции?
Целостность данных в классе `Hospital` поддерживается через:
1.  **Строгую типизацию**: метод `add()` проверяет, является ли объект экземпляром класса `Patient` через `isinstance`.
2.  **Запрет дубликатов**: перед добавлением проверяется, нет ли уже такого пациента в списке (используется метод `__eq__` из ЛР-1).
3.  **Валидацию индексов**: метод `remove_at()` проверяет, входит ли индекс в допустимый диапазон, предотвращая падение программы.

### Вопрос 3. Как реализована итерация и доступ по индексу?
Для интеграции со стандартным поведением Python реализованы магические методы:
- **`__iter__`**: позволяет использовать больницу в циклах `for patient in hospital`.
- **`__len__`**: позволяет получать количество пациентов через `len(hospital)`.
- **`__getitem__`**: позволяет обращаться к пациентам по индексу через квадратные скобки `hospital[0]`.

### Вопрос 4. Какие критерии поиска и фильтрации выбраны?
В коллекции реализованы методы для удобной работы с данными:
- **Поиск**: `find_by_name(name)` возвращает первый найденный объект или `None`.
- **Фильтрация**: `get_sick_patients()` и `get_healthy_patients()` возвращают новые списки, отфильтрованные по статусу здоровья.
- **Сортировка**: `sort_by_age()` возвращает список пациентов, отсортированный от младшего к старшему.
